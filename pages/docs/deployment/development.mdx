# Development Environment Variables

This document provides specific configurations for the development environment of Eyygo projects, building upon the base environment variables.

## 📋 Sample .env File for Development

```dotenv
NODE_ENV=development
WS_PORT=3333
HTTP_PORT=8000
ALLOWED_ORIGINS=localhost:3000
CERT_FILE=
KEY_FILE=
LOG_FILE=server.log
DEBUG=true
TIME_ZONE=Asia/Singapore
DB_ENGINE=sqlite3
DB_NAME=db.sqlite3
DB_USER=
DB_PASSWORD=
DB_HOST=
DB_PORT=
SECRET_KEY="dev_secret_key_change_me"
```

## 🔧 Development-Specific Configurations

### 🌍 Application Environment

#### NODE_ENV
- 🔢 **Value**: `development`
- 💡 **Tip**: Ensures the application runs in development mode with appropriate debugging and logging.

### 🚪 Ports

#### HTTP_PORT
- 🎯 **Example**: `8000`
- 💡 **Tip**: Choose a port that doesn't conflict with other local services.

### 🔒 Security Settings

#### ALLOWED_ORIGINS
- 🎯 **Example**: `localhost:3000`
- 💡 **Tip**: Include all local development URLs that will access your API.

#### CERT_FILE and KEY_FILE
- 💡 **Tip**: Typically left blank for local development unless testing HTTPS.

#### SECRET_KEY
- 🎯 **Example**: `"dev_secret_key_change_me"`
- 💡 **Tip**: Use a dummy value for development, but ensure it's different from production.

### 📊 Logging and Debugging

#### LOG_FILE
- 🎯 **Example**: `server.log`
- 💡 **Tip**: Keep it simple for local development.

#### DEBUG
- 🔢 **Value**: `true`
- 💡 **Tip**: Enables verbose logging for development purposes.

#### TIME_ZONE
- 🎯 **Example**: `Asia/Singapore`
- 💡 **Tip**: Set to your local time zone for accurate timestamps during development.

### 🗄️ Database Configuration

#### DB_ENGINE
- 🔢 **Value**: `sqlite3`
- 💡 **Tip**: 
  - SQLite is ideal for local development on Ubuntu or Mac.
  - Use `mysql` or `postgresql` for Windows or more complex setups.

⚠️ **WARNING**
SQLite3 is not working on Windows, use MySQL or PostgreSQL instead.

#### DB_NAME
- 🎯 **Example**: `db.sqlite3`
- 💡 **Tip**: For SQLite, this is a file path.

#### DB_USER, DB_PASSWORD, DB_HOST, DB_PORT
- 💡 **Tip**: Leave blank for SQLite. Fill in appropriate values for MySQL or PostgreSQL.

## 🚀 Getting Started

1. Copy the sample `.env` file to your project root.
2. Modify the values according to your local development setup.
3. Ensure the `.env` file is added to your `.gitignore` to prevent committing sensitive information.

Remember, these settings are for development only. Never use development environment variables in production!
